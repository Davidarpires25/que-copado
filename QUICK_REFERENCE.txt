│  ARCHITECTURE:                                                              │
│                                                                              │
│  ┌─ Your Next.js Cloud ─────────────────────────────────────────────┐      │
│  │                                                                   │      │
│  │  API: POST /api/orders (create order)                            │      │
│  │       ↓ calls createPrintJob()                                   │      │
│  │       ↓ inserts rows into print_jobs table                       │      │
│  │                                                                   │      │
│  │  API: GET /api/print-queue?restaurant_id=X                       │      │
│  │       (agent polls this every 500ms)                             │      │
│  └─────────────────────────────────────────────────────────────────┘      │
│            ↓                                                                 │
│  ┌─ Restaurant Network ─────────────────────────────────────────────┐      │
│  │                                                                   │      │
│  │  Node.js Print Agent (runs in Docker)                            │      │
│  │  • Polls /api/print-queue every 500ms                            │      │
│  │  • Formats ESC/POS commands                                      │      │
│  │  • Sends to TCP:9100 on each printer                             │      │
│  │  • Marks jobs as success/failed                                  │      │
│  │                                                                   │      │
│  │  Printers (network-connected):                                   │      │
│  │  • Kitchen: Epson TM-T88V (80mm) @ 192.168.1.10:9100            │      │
│  │  • Receipt: Epson TM-T88V (80mm) @ 192.168.1.11:9100            │      │
│  │  • Bar (optional): Star TSP143 @ 192.168.1.12:9100              │      │
│  │                                                                   │      │
│  └─────────────────────────────────────────────────────────────────┘      │
│                                                                              │
│  CODE EXAMPLE:                                                              │
│                                                                              │
│  // When order is placed, create print jobs                                │
│  await createPrintJob({                                                     │
│    order: newOrder,                                                         │
│    restaurantId: 'rest-123',                                                │
│    printKitchen: true,    // → Kitchen Printer                             │
│    printReceipt: true     // → Receipt Printer                             │
│  });                                                                        │
│                                                                              │
│  // Agent (runs on restaurant server) polls and prints                      │
│  setInterval(async () => {                                                  │
│    const jobs = await GET /api/print-queue?restaurant_id=rest-123          │
│    for (const job of jobs) {                                                │
│      const socket = net.createConnection(job.printer.ip, 9100)             │
│      socket.write(job.escpos_data) // ESC/POS commands                     │
│      // Mark as success                                                    │
│    }                                                                        │
│  }, 500)                                                                    │

Key Takeaways:
  1. ESC/POS is the universal printer language (30+ years old, stable)
  2. Network printers are industry standard (Epson TM-T88V, Star TSP143)
  3. Local Node.js Agent is the best fit for your Next.js SaaS
  4. 500ms-2s latency is acceptable for kitchen (better is nice, not critical)
  5. Scales to 10,000 restaurants with zero licensing costs
  6. Total cost of ownership is 2-3x lower than alternatives at scale

Remember: Fudo doesn't win on printer technology. Fudo wins on delivery
integration. Build printing competently, then focus on restaurant management
and delivery aggregation features that actually differentiate your product.
